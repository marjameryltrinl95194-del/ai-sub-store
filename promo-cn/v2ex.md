## 帖子 3：技术向

**标题：搞了个 AI 订阅代购站的技术架构分享**

**正文：**

分享一下我搭 AI 订阅代购站（AISubHub）的技术栈，给想入行的参考：

**前端：**
- 纯 HTML + CSS + JS，零框架
- 暗色紫色主题，响应式设计
- 8 国语言（data-i18n 属性驱动，含阿拉伯语 RTL）
- 部署在 Cloudflare Pages

**支付：**
- USDT TRC20 / BEP20 / Polygon
- 链上交易验证（暂未对接链上 API，人工确认后发货）

**产品：**
- ChatGPT Plus / Pro
- Claude Pro / Max
- Gemini Pro

**值得分享的点：**
1. 多语言用 data-i18n 属性 + 一个 i18n 对象实现，比引入框架轻量很多
2. 支付流程是类 Web3 风格 — 展示地址 → 用户转账 → 链上确认 → 发卡
3. 纯静态部署，零后端成本，适合 MVP 验证

**踩过的坑：**
- captcha 要在服务端做才对，纯前端验证码等于没有（当前方案仅供展示）
- 支付地址展示期 15 分钟倒计时，这个必须前端做

有什么问题欢迎交流。
